## Props

- `object` (`T`) — Value to traverse.
- `path` (`ValidPath<T, Path>`) — A dot path or explicit segment tuple. Literal
  paths are checked against `T`; a widened `string` or `PropertyKey[]` is
  accepted as a dynamic path.
- `defaultValue` (`Default`) — Returned when the path is missing or resolves to
  `undefined`. The third argument is required by the fallback overload; omitting
  it always keeps `undefined` in a possibly missing result.

### Returns

`GetPathResult<T, Path, Default>` — The value inferred at a literal path.
Optional or union branches include `undefined`; a fallback replaces that
`undefined` branch. Dynamic paths return `unknown`.

Only provably single string or number literals and unique symbols are exact
segments. Broad, union, branded, and template-pattern keys—and unions of tuple
shapes—are dynamic, avoiding a specific result for a runtime-selected path.

Every segment must be an own property at runtime. An empty path returns the root,
and explicit segment tuples support symbols and numeric array indexes. Array
indexes range from `0` through `4294967294`; larger digit-only keys are ordinary
properties and must be declared on the array type to use them as literal paths.
Explicitly typed own array metadata keys are supported, while inherited methods
such as `map` and `push` are not valid literal paths.

Known built-in prototype members are excluded for the same reason: for example,
`getTime` is not an own path on a `Date`, and `size` is not an own path on a
`Map`. Explicitly declared custom own properties on those values remain
available.
