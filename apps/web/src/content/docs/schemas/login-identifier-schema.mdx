## Props

- `input` (`string`) — An email address or username. Outer whitespace is
  removed once before either format is evaluated.

### Returns

`LoginIdentifier`, represented as `Email | Username`.

Email values preserve their casing and have a maximum length of 254.
Username values are lowercased, normalized to NFC, limited to 3–30 Unicode code
points, and restricted to the same alphanumeric and separator policy as
`usernameSchema`. Invalid input reports one login-identifier error rather than
requiring the caller to choose a branch.
